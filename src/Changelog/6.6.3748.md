## ⚠️ Изменения для Enterprise клиентов

В связи с переходом с Spring Boot 2 на Spring Boot 3, для установки новой версии необходимо:

### Изменить переменные окружения

**1. PATH**

```bash
# Было (Java 11)
/usr/local/openjdk-11/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin

# Стало (Java 21)
/opt/java/openjdk/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
```

**2. JWTSECRET**

Новая версия требует более строгой валидации ключа. Токены подписываются алгоритмом **HS512**,
которому нужен ключ **не короче 512 бит — минимум 64 символа**. С коротким ключом приложение
отклонит вход пользователя с ошибкой `WeakKeyException` в логах.

Проверка длины:
```bash
echo -n "$JWTSECRET" | wc -c   # должно быть ≥ 64
```

Генерация нового ключа (если текущий короткий):
```bash
openssl rand -base64 64 | tr -dc 'a-zA-Z0-9' | head -c64
```

### Чеклист обновления

- [ ] Обновить PATH (если явно задан)
- [ ] Проверить длину JWTSECRET (≥64 символа)

## ✨ Новые возможности

- Появилось управление AI‑подсказками в настройках администратора: теперь можно централизованно настраивать, какие подсказки и сценарии использует ИИ‑помощник в компании.  
- Добавлен экспорт реестра: вы можете выгружать данные реестра для анализа, отчетности или переноса в другие системы.  
- Реализован перенос оргструктуры (orgchart transfer): стало проще переносить и обновлять структуру подразделений между средами или версиями.  
- В моделере появился новый бейдж для DES‑моделей: их легче отличать от обычных BPMN‑диаграмм и быстрее находить нужный тип моделей.  

## 🚀 Улучшения

- Обновлён хедер в дереве реестра и добавлена кнопка «раскрыть/свернуть всё», а само дерево по умолчанию свернуто (если нет сохранённых настроек) — работать с большими структурами стало проще и нагляднее.  
- Улучшено отображение дерева и уровней оргструктуры: элементы больше не «разъезжаются», фильтры работают согласованно, структура читается стабильнее.  
- Окно сохранения процесса открывается заметно быстрее, а его состояние корректно обновляется — экономит время при частых правках моделей.  
- В календаре назначений улучшена работа с данными (json‑формат), что делает планирование и просмотр загрузки исполнителей надёжнее.  
- На маркетинговых и справочных страницах (Help, элементы BPMN, DES) оптимизированы блоки с призывами к действию и добавлены «скелетоны» при загрузке — страницы воспринимаются быстрее и лучше подталкивают к нужному действию.  
- В блоге улучшены баннеры и UTM‑метки: переходы из статей теперь лучше отслеживаются, а полезные действия пользователей — выше.  
- Страница оптимизаций переработана: добавлены новые секции с признаками проблем и реальными кейсами, улучшена структура, вкладки и глоссарий — легче понять, какие улучшения процессов возможны и как их применять.  
- Улучшена навигация и удобство чтения: добавлены вкладки для оптимизаций и глоссария, расширенные карточки, двухколоночный глоссарий, блок «С чего начать» и кнопка «Наверх» в более удобном месте.  
- Улучшена доступность и SEO справочных материалов (HowTo и FAQ): пользователям проще находить ответы через поиск и ориентироваться в документации.  
- В интерфейсе тарифов добавлен «скелетон» для блока текущего плана — страница выглядит аккуратнее во время загрузки.  

## 🔧 Исправления

- Улучшена работа доступа к расшаренным диаграммам: неавторизованные пользователи больше не получают лишние права, а ответы системы на отсутствие доступа стали корректнее.  
- Исправлены ошибки при импорте по категориям и скорректировано имя файла при экспорте — теперь операции с данными проходят стабильнее и понятнее для пользователя.  
- Уточнены проверки и валидация при создании администратора и сохранении процессов: кнопка сохранения блокируется при ошибках, а некорректные данные отлавливаются до записи.  
- Удалён устаревший платёжный провайдер, добавлены проверки для рекуррентных платежей и исправлены ошибки с типом оплаты — оплата и продление подписки работают надёжнее.  
- Исправлены проблемы с отображением карточек DES, иконок и нижних границ табов, а также мелкие визуальные огрехи — интерфейс выглядит более аккуратно и предсказуемо.  
- Устранены рывки и «скачки» при прокрутке статей, а также лишние или дублирующиеся поля поиска — чтение материалов и поиск нужной информации стали комфортнее.  
- Исправлена работа поиска по нескольким авторам в устаревшем методе, а также ряд неточностей в фильтрах и формировании данных форм — результаты поиска и формы ведут себя ожидаемо.  
- Улучшена совместимость аналитики и работы интерфейса в Safari, чтобы отчёты и страницы открывались без сбоев.  
- Скрыты внутренние технические коды оптимизаций и упоминания «бесплатно» в некоторых кнопках и текстах — интерфейс стал чище и понятнее для конечных пользователей.
