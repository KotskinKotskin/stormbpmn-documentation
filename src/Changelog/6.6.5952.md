# Версия 6.6.5952 от 09.06.2026

::: danger ⚠️ ВАЖНО ПЕРЕД ОБНОВЛЕНИЕМ — требуется PostgreSQL с расширением pgvector
Начиная с этой версии приложению **обязательно** нужна СУБД PostgreSQL с установленным расширением **`pgvector`** — оно используется базой знаний AI‑чата и векторным поиском.

**Без расширения приложение не запустится.** На старте миграция базы данных выполняет `CREATE EXTENSION vector`; если расширения нет, контейнер аварийно завершается с ошибкой — режима «мягкой деградации» не предусмотрено.

**Что сделать перед обновлением:** доставить расширение `pgvector` в вашу PostgreSQL. Есть несколько способов — с командами и важными нюансами они собраны в отдельном гайде:

➡️ **[Установка pgvector в PostgreSQL](../recipes/Pgvector_setup.md)**

Требование появилось вместе с базой знаний AI‑чата. Версии **до `6.6.5694` включительно** расширение не требовали — при обновлении с них смена образа/настройки PostgreSQL обязательна.
:::

## ✨ Новые возможности

- В моделировании процессов появилась панель результатов симуляции с наглядным обзором ключевых KPI, «здоровьем» процесса и списком проблем — теперь проще понять, где процесс работает хорошо, а где теряются время и ресурсы.  
- Добавлена 2×2 KPI‑сетка и индикатор «здоровья» процесса (дуга с подоценками) по итогам симуляции, что помогает быстро увидеть узкие места и приоритизировать улучшения.  
- В редакторе диаграмм доступен новый вид представления данных в виде «chip‑pivot» с возможностью переключаться на плоский список — это упрощает анализ сложных моделей и связанных объектов.  
- Встроенный виджет бронирования встреч теперь работает на Storm‑CRM вместо стороннего сервиса: клиенты могут записываться прямо с сайта, а все заявки сразу попадают в вашу CRM.  
- В интерфейсе моделирования улучшены инструменты точного управления геометрией элементов и авто‑раскладкой диаграмм, что позволяет быстрее приводить схемы к аккуратному и читаемому виду.  
- В админ‑панели появилась возможность восстанавливать роли и элементы оргструктуры, если они были удалены или повреждены, что снижает риск потери важных настроек доступа.

## 🚀 Улучшения

- В AI‑чате и модуле работы с диаграммами стало проще откатываться к предыдущим версиям схем: это повышает безопасность экспериментов с моделями и ускоряет возврат к проверенным вариантам.  
- Маркетинговые страницы и формы (включая подсказки по встраиванию форм/iframe) стали более гибкими по оформлению и лучше вписываются в фирменный стиль компании.  
- Авто‑раскладка BPMN‑диаграмм стала аккуратнее и предсказуемее, что уменьшает ручную доработку схем после автоматического размещения элементов.  
- Процессы симуляции можно запускать, отслеживать статус в реальном времени и быстро перезапускать на основе предыдущего запуска, что ускоряет цикл «смоделировал — посмотрел результат — улучшил».  

## 🔧 Исправления

- Устранены ситуации, когда при повторной загрузке одних и тех же данных через публичный API могли появляться дубликаты элементов оргструктуры.  
- Улучшена работа фильтрации и продления подписок: ручные счета и переход с пробного периода на платный теперь обрабатываются корректнее, без неожиданных сценариев для клиентов.  
- Повышена стабильность AI‑чата: уменьшено количество ложных отказов в ответах и улучшен поиск по базе знаний, чтобы нужные материалы находились по ключевым словам.  
- Исправлены редкие случаи «тихой» потери связей при пакетных операциях с ролями и элементами оргструктуры, что повышает надёжность администрирования.  
- Повышена устойчивость интеграции с внешним AI‑коннектором: перезапуски клиентских приложений больше не приводят к сбоям в работе сценариев.

Все Docker образы криптографически подписаны с помощью [Sigstore Cosign](https://docs.sigstore.dev/). Для проверки подлинности образа:

**1. Установите Cosign:**
```bash
# macOS
brew install cosign

# Linux
curl -sSL https://github.com/sigstore/cosign/releases/latest/download/cosign-linux-amd64 -o /usr/local/bin/cosign
chmod +x /usr/local/bin/cosign
```

**2. Проверьте подпись:**
```bash
cosign verify \
  --certificate-identity-regexp="https://github.com/KotskinKotskin/stormfullstack/*" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  cr.selcloud.ru/stormbpmn-enterprise/stormbpmn_fullstack_enterprise:6.6.5952
```

Успешная проверка гарантирует, что образ собран в официальном CI/CD и не был изменён.
